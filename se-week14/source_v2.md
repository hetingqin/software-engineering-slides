## 第一讲：Agent Skills — AI 的"职业技能包"

### 1. 什么是 Skills？
如果把大语言模型 (LLM) 和基于它的 Agent（如 Claude Code）比作一个“刚入职的绝顶聪明的本科生”，那么 **Skills 就是他上岗所需的标准操作手册（SOP）**。

*   **没有 Skill 时**：你让 AI “去把那篇 Markdown 转成精美的企业级 PPT”。AI 会盲猜怎么做，极速乱编一堆 Python 库去试图生成，最后因为没有设计模版、没有配色标准而生成出几张惨不忍睹的图。
*   **有 Skill 时**：它严格阅读 `SKILL.md` 指令，按照设定的“图文配比规则”、“16种视觉风格方案”去调用外挂工具，生成极高水准的成品。

### 2. 深入理解 Skill 架构（以 slide-deck 为例）
一个标准的现代工程化 Skill 通常包含：
*   **SKILL.md**：系统的核心大脑文件，全用**自然语言**编写的操作守则（Workflow）。
*   **scripts 文件夹**：封装好的底层硬编码执行器（因为 AI 调现成脚本比自己从头写原生代码更稳）。
*   **references / examples**：给 AI 当参照物的模版。

**核心意义**：
在旧时代，我们把复杂逻辑封装成“类库/函数库”，供其他程序员**调用接口**。
在新时代，我们将复杂逻辑封装成“Skills 包”，供强大的 AI 读完后听从**自然语言的召唤**。接口变了，但“抽象与封装”的软件工程灵魂没有变。

### 3. AI 工程生态中的套娃戏法
*最高级的玩法：基于 Agent 操作系统，让人类写 Skill 的时代也快结束了。*
开发流程演变为：**人类口述宏观流程 → AI 自己按规范写出 `SKILL.md` → 人类审核微调 → 以后都由其他 AI 端点去读取并执行这个 Skill 本身**。人真正跳出了拧螺丝的循环。

## 第二讲：被放大的质量灾难与代码审查防御

### 1. Vibe Coding 繁荣背后的暗礁
虽然 AI 写 CRUD 的效率提高了 55%，但学术界对全行业超千万量级 copilot 生成代码的统计结果触目惊心：

*   **变动率激增（Code Churn）**：AI 写的代码缺乏远期规划，往往第二天就要被人类或者另一套 AI 指令推翻重写。
*   **重构消亡**：因为写得太快，几乎无人愿意再花时间精雕细琢去重构糟糕的设计，代码直接沦为“只求跑通的屎山”。
*   **安全黑洞**：在 Python 这类语言中，AI 直接生成的代码含有安全漏洞（如未经参数化的 SQL 注入、写死的密钥假数据）的概率高达 **29.1%**。

### 2. 构建新时代的自动化防线
既然源头的代码产出不再是瓶颈，如何接稳这海量的不受控代码？

*   **AI Code Review（以毒攻毒）**：不仅在人与人之间互相 Review。提交代码前挂载一个苛刻的专注挑刺的 Agent（如 Cursor 审查模式）先做第一轮阻截。
*   **左移安全监控（Shift Left）**：在 CI/CD 流水线（如 GitHub Actions）中必须前置嵌入彻底的红线静态扫描工具。哪怕是 AI 写的，不达标也 100% 打回重写。
*   **强制测试覆盖率**：不要自己写测试，指派另一个独立会话的 AI 根据需求文档反向写测试用例去检验上一个 AI 生成的源码，达成博弈。

## 第三讲：洗牌重生的新时代工程师能力模型

当最基础的搬砖写代码不再是核心壁垒，“我为什么比 AI 更值钱”？

### 1. 贬值与淘汰的旧技能
*   **肌肉记忆型** API 默写能力。
*   在不同语言间来回切换的“纯语法熟练度”。
*   枯燥的正则表达式手搓能力、繁复的建表 SQL 配置。

### 2. 升值的新版“五维能力模型”
**旧模型**：编码实现 + 调试纠错 + 框架精通
**新模型** 👇：

1.  **架构设计与业务判断力（核心！）**：AI 可以瞬间写出 5 个文件，但如果你一开始判断错了系统该微服务还是单体，或者该功能其实不是客户痛点，代码写得再快也是制造灾难废品。这也是 AI 目前绝对无法替代的领域。
2.  **Prompt 工程能力**：把“自然语言”当成地球上最严谨的编程语言。能清晰表达上下文、约束条件和反面案例，是驾驭 L3 级别 Agent 的钥匙。
3.  **AI 协作与鉴毒能力**：不要把 AI 当神，把它当一个不知疲倦、聪明绝顶但极度马虎且有幻觉的“大一新生实习生”。你的角色是 Technical Lead（技术负责人），你需要一眼看穿它生成的代码哪里可能会埋下系统崩溃的地雷。
4.  **工程纪律的守夜人**：“代码未动，计划先行”。逼迫自己和 AI 必须先生成文档和 UML 构图，强忍住 Vibe Coding 的快感，维持住了项目的长期可读性。
5.  **扎实的计算机底层理论**：遇到各种依赖爆炸或是诡异堆栈溢出，AI 开始原地绕圈子乱答时。只有你凭借操作系统、网络协议栈与设计模式的功底把它踹开亲自排雷。这就叫“**擦屁股的艺术**”。

## 课后讨论与总结陈词

### ✨ 结语箴言
其实所有的技术变革：从 C 到 Java 到 SpringBoot 到 Kubernetes 云原生，再到今天的 AI Agent。都只有一个最核心的诉求：**隐藏底层的繁杂，让人类更专注于创造真正的业务价值。**

计算机技术几个月就天翻地覆，但如《人月神话》开篇所说，解决复杂系统的“道”——**软件工程背后的本质并没有变**。
无论你是操作穿孔纸带、用键盘敲击 Java 还是用麦克风向 Agent 说话，**“把事情当作项目来推进，用系统化工程方法控制风险、保证质量并解决问题”** 的思维，才是你立命的最终护城河。
