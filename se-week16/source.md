### 3. 从学生到工程师的思维转变

| 学生思维 | 工程师思维 |
|----------|------------|
| 完成作业就好 | 追求质量和可维护性 |
| 一个人写完 | 团队协作 |
| 只关心功能 | 关心性能、安全、可用性 |
| 写完就扔 | 持续维护和迭代 |
| 代码能跑就行 | 代码是给人读的 |
| 技术至上 | 技术服务于业务 |

### 4. 送给大家的话

> **"万事皆项目"**
>
> 软件工程不仅适用于软件项目，也适用于生活中的一切。
> 用项目视角看问题，用工程方法解决问题，站在整体而非局部。
>
> 无论技术如何变化——从汇编到高级语言，从瀑布到敏捷，从手写代码到 AI 编程——
> **软件工程的"道"始终不变**：
>
> 用系统化的、规范的、可度量的方法，创造有用的产品。
